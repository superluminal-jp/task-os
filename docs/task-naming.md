# Task Naming Convention

タスク名の命名規則。Notion Board（Focus Areaグルーピング）での視認性を基準とする。

---

## フォーマット

```
{対象} {アクション}
```

- **対象**: 何についてのタスクか（機能名・システム名・ドメイン名）
- **アクション**: 何をするか（体言止め）
- 折り返し幅（全角17文字）を意識して対象とアクションが自然に2行に分かれるように書く
- 字数は強制しない

### 例

```
ログ配信 リソースポリシー追加
APIスロットリング 方針決定
ダッシュボード 承認取得・全庁公開
校正アプリ ログ確認・改善方針決定
```

---

## 固有名詞の扱い

**人名・組織名は使わない。** 行為の対象・役割・機能名に置き換える。

| 固有名詞の種類 | 変換方針 | 例 |
|---|---|---|
| 人名 | 役割または行為で表現 | `山口参事官の承認` → `承認取得` |
| システム固有名（英語） | 機能名に変換 | `compound_splitter` → `分割処理`, `CloudWatch Logs` → `ログ配信` |
| サービス名 | 機能名に変換 | `Cloud Run` → `デプロイ確認`, `ChangeSet` → `デプロイ設定` |
| spec番号・タスクID | そのまま使用（識別に必要） | `spec-005`, `T001–T021` |

---

## Focus Area との関係

Focus Area はボードのグルーピング軸なので、タスク名に含めない。  
Project・Work Item も別プロパティで管理するため名前には不要。

タスク名が伝えるのは **「対象 × アクション」のみ**。
